Moar about AngularJS


  • Initially released in 2009 by Google
  • Used to create Single Page Applications (SAP)
  • A framework for adding interactivity to HTML
  • Falls within the family of MV* frameworks, and is often compared to Ember.js, Backbone.js, & React

What is a MV* framework?

  • Main features are some combination of routing, data binding, templates/views, models & data access
  • Can be client- or server-side, and not specific to JavaScript
  • A variation of an MVC (Model View Controller)
  • Model — a type or piece of data to display or manipulate (a user, an account, a puppy, etc)
  • View — the markup & templates for the User Interface
  • Controller — handles input & updates the state of model

What is a MV* framework?

Image Credit: Wikimedia

What type of MV* is Angular?

  • There's much debate on what exactly Angular is
  • Angular has evolved since release, and developers implement it differently
  • Many agree (including Angular!) that it is an MVW (Model View Whatever)

Why Use Angular?

  • Easy to get up and running
  • Code organization
  • Speedy websites
  • Unit testing
  • Many, many resources

Directives: The Heart and Soul of Angular

  • Directives are the feature of Angular that make it truly powerful
  • "At a high level, directives are markers on a DOM element (such as an attribute, element name, comment or CSS class) that tell AngularJS's HTML compiler... to attach a specified behavior to that DOM element... or even to transform the DOM element and its children."
  • Angular comes with a bunch of built-in Directives

Built-In Directive: ng-app

ng-app is a required directive used to bootstrap our AngularJS application designate the root or parent element 

<body ng-app>
  <h1>Hello World!</h1>
</body>

Expressions

  • Angular expressions are JavaScript-like code snippets that are mainly placed in interpolation bindings
  • A way to insert data into html via a JavaScript string, number, object, or array

Two-Way Data Binding

The ng-model directive binds the value of an input, select, or textarea to an object that can be used in an expression

<body ng-app>
  <label>Name:</label>
  <input type="text" ng-model="yourName">
  <hr>
  <h1>Hello {{yourName}}!</h1>
</body>

Two-Way Data Binding

Expressions can be inserted as the value html attributes as well as text within html tags

<body ng-app>
  <label>Name:</label>
  <input type="text" ng-model="user.status">
  <hr>
  <h1 class="status-text-{{user.status}}">Your status is:  {{user.status}}!</h1>
</body>
